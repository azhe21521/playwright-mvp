# 代码规范

## 语言使用
- **中文**: 回答问题、文档、代码注释
- **英文**: 代码本身（变量名、函数名、类名等）

## 编码习惯
- 使用 TypeScript 类型注解
- 使用 `const` 定义引用；必须重新赋值时用 `let`；禁止 `var`
- 使用字面量语法创建对象和数组，不用 `new` 关键字；声明对象时简写属性放在前面
- 使用单引号 `''` 定义字符串；禁止跨行字符串连接符；构建字符串用模板字符串，不用拼接
- 文件 200-400 行为宜，800 行封顶
- 函数不超过 50 行，嵌套不超过 4 层


## 注意事项
- 遵循项目现有代码风格和命名约定
- 避免过度设计和不必要的抽象
- 不清楚的信息应提问或记录为 TODO，不要根据经验想象
- 尽量忽略错误日志及事件上报逻辑，避免上下文信息冗杂
